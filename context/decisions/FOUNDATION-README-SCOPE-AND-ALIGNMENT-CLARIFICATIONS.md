# FOUNDATION-README-SCOPE-AND-ALIGNMENT-CLARIFICATIONS

## CONTEXT

`phases/backlog/foundation/README.md` was reviewed and found to understate the actual foundation contract, to frame CEO / CTO / DEV too loosely, and to imply a predeclared later foundation layer that the current workplan does not actually promise.

## OPTIONS

- Leave the README narrow and ambiguous.
- Clarify it so it matches the live foundation contract and the approved CEO / CTO / DEV future-alignment requirement.

## DECISION

`phases/backlog/foundation/README.md` must describe foundation as including:
- folder structure
- manual context mechanism
- foundation governance rules
- agent bootstrap and routing surfaces

CEO / CTO / DEV is a protected mandatory future-alignment requirement.
Future system-definition documents and implementation must align to that structure.
The README must not overclaim that the full role layer is already specified in detail.

The README must use gate-based language:
- if foundation is incomplete after the current step, define the next foundation step
- otherwise leave foundation and move to the next workplan layer
