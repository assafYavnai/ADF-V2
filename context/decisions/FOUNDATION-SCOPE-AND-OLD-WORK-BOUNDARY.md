# FOUNDATION-SCOPE-AND-OLD-WORK-BOUNDARY

## CONTEXT

The foundation layer needed a clear boundary before broader ADF-V2 definition work could continue.
The discussion focused on whether foundation should already define vision, mission, scope, and architecture, and on how the old work should be treated while foundation is being created.

## OPTIONS

- Make foundation broad and include system-defining documents such as vision, mission, scope, and architecture.
- Keep foundation narrow and use it only to establish folder structure and the manual context mechanism.
- Treat old work as active design input during foundation creation.
- Abandon old work as active context for foundation and prevent agents from pulling it back in by default.

## DECISION

Foundation is narrow.
For this stage, foundation means only:
- folder structure
- context mechanism

System-defining documents such as vision, mission, scope, architecture, role definitions, and workflows are deferred to later work.

For foundation work, the old work is abandoned as active context.
Agents must not point back to it concretely or treat it as part of the active truth surface unless the CEO explicitly chooses to bring it in for clarification.
